# 設計方針

## なぜ作ったか

スポーツの練習・試合で使うタイマーとして作った。既存の汎用タイマーアプリではなく自作した理由は、スポーツの現場特有の要件があったから:

- **コートの端やベンチなど、スマホから離れた場所からでも残り時間を確認したい** — 一般的なタイマーアプリの数字は小さく、離れると読めない
- **スマホを地面やベンチに置きっぱなしにする間、画面ロックを解除したままにしたくない** — 誰でも中身を見られる・操作できる状態のスマホを長時間放置するのは避けたいが、タイマーの停止・時間調整はその場にいる誰でもすぐできてほしい
- **練習中に手が離せない・とっさに時間を伸ばし縮めしたい場面がある** — 複雑な設定画面を都度開くのではなく、動かしたまま片手で触れる最小限の操作にしたい

この3つの要件が、そのまま以下の設計方針になっている。

## 設計方針と実装の対応

### 1. どこからでも読める大画面表示

`CountdownDisplay` (`src/components/CountdownDisplay.tsx`) は、画面の余白ではなく利用可能な領域いっぱいに数字を表示することを最優先にしている。固定フォントサイズではなく、実際のレイアウト計測結果から画面に収まる最大サイズを逆算して描画する(`FIT_SCALE`で少し余裕を持たせているが、基本方針は「入る限り最大化」)。縦持ち・横持ちのどちらでも同じロジックでフィットするため、スマホをどう置いても遠くから読める大きさが確保される。

`TimerScreen` 側でも、コントロール類(調整ボタンなど)は画面下端に薄く配置するだけで、数字表示エリアを画面のほとんどの高さに割り当てている。

### 2. ロック画面上に表示し続け、誰でもすぐ操作できる

`plugins/withShowOverLockscreen.js` という Config Plugin で、`MainActivity` に以下の3つの Android ウィンドウフラグを付与している:

- `FLAG_SHOW_WHEN_LOCKED` — ロック画面の上にアプリ画面をそのまま表示する
- `FLAG_TURN_SCREEN_ON` — 画面が消えていても即座に点灯させる
- `FLAG_KEEP_SCREEN_ON` — スリープで画面が消えるのを防ぐ

これはアラーム時計アプリと同じ仕組みで、「スマホのロックを解除したまま放置する」ことと「誰でもすぐ画面を見て操作できる」ことを両立させるための選択。ロックは掛かったまま(=スマホの中身は保護されたまま)でも、タイマー画面だけはその上に出続けて誰でも止められる。`useKeepAwake()` (`expo-keep-awake`) も合わせて使い、アプリがアクティブな間は端末がスリープしないようにしている。

### 3. 片手で完結する最小限のUI

- 設定画面 (`SetupScreen.tsx`) は、時間を選ぶホイールピッカーと大きな「スタート」ボタンのみ。詳細設定(アラーム音量)は歯車アイコンの奥に隠し、通常操作の邪魔にしない
- タイマー実行中 (`TimerScreen.tsx`) は、±10秒/±30秒/±1分の調整ボタンと一時停止・リセットのみを画面下部に配置。メニューを開かずその場でワンタップで時間を伸縮できる
- タイマー完了後は画面全体が反転点滅するシンプルな停止画面になり、画面のどこをタップしても止まる(狙って小さいボタンを押す必要がない)

### 4. 確実に気づけるアラーム

音が小さい・消音になっているとスポーツ中は気づけないため、タイマー完了時は端末のメディア音量を設定した音量まで強制的に引き上げてから鳴らす(自作ネイティブモジュール `modules/volume-control`)。停止した時点で元の音量に復元するため、常用の音量設定を壊さない。

### 5. 時間のズレを起こさない実装

`useCountdown` (`src/hooks/useCountdown.ts`) は、残り時間をカウントダウンする値としてではなく「終了時刻(壁時計時刻)」として保持している。画面ロックやバックグラウンド中にJSタイマーが間引かれても、フォアグラウンド復帰時に壁時計との差分を取り直すだけで正確な残り時間に復元できる。スポーツ現場でスマホをポケットに入れる・画面が一時的に消えるといった状況でも、タイマーの精度が崩れないようにするための選択。
